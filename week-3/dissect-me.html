Git, GitHub, and the Command Line

August 23, 2015

My first week at DBC is almost complete and I've learned a ton. For my own sake, and the sake of you, dear reader, I am going to walk through the basics of git and GitHub, as well as some git commands you'll need to use in the command line.


Git is an incredibly handy open source version control system. Sooooooooo... what is that? Version control software allows you to save and record many versions of a project as you work on it, and access those different versions whenever you like. You have control of all of those versions. Get it? Version. Control. Version control!

This is obviously very handy for any project where you might make a mistake or bad choice and want to go back, but it is especially handy for coding, where one rogue character can mess up an entire project. With git you can just go back to a working version and try again.


Git helps you keep track of changes with commit messages and the git log. With each submitted change, or commit, a good programmer submits a commit message that explains exactly what was done. These commits and their messages are always accessible via the git log, which shows every commit since the initial one. Pretty nifty!


Github is an awesome way to store, track, collaborate on, and change your code. There are many benefits to using GitHub. For one, your work and all of your commits now exist online, which will save you if your computer crashes. The interface is excellent, and accessing your commits is incredibly easy. The interface also allows you to see exactly what was changed when you submit changes by using a helpful red/- and green/+ comparison. But perhaps GitHub's greatest strength is that it allows you to work collaboratively with people around the world on your work and theirs. GitHub is a social networking site (like Facebook), and if you like, you can make your work public, allowing people to copy, change, and add to your work. All of this exists at GitHub.com. Check it out!


Here is my nifty guide to making changes to git/GitHub projects. The command line commands will only work if you have git set up, and this will be super confusing to true beginners. Come back once you've played around in git and GitHub a little bit!
  1. Navigate to your repository in the command line using the cd command line
  2. Create a branch: git co -b new-branch-name
  3. Make changes
  4. Check your git status: git status
  5. Add your files and changes to be committed: git add . (or the specific file name)
  6. Check your git status: git status
  7. Commit your changes and add a message: git commit -m "message"
  8. Check your git log: git log
  9. Navigate back to the master branch: git co master
  10. Pull the current master branch (in case a team member has made changes): git pull
  11. Navigate back to your new branch: git co new-branch-name
  12. Merge it with the master branch: git merge master
  13. Push your changes up to GitHub: git push origin new-branch-name
  14. Navigate to github.com
  15. Click pull request
  16. Confirm merge
  17. Delete the old branch (this is good organizational practice): git branch -d new-branch-name
